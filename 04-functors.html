Why This Matters for Functional Programming

Functors let you translate between different "worlds" of computation. This is the key to handling effects and different computational models.

Generic types are functors: When you write `List` or `Option`, you're using a functor. The functor maps types to types. If you have an `Int`, the functor maps it to `List Int`. If you have a function `f : Int → String`, the functor maps it to `List Int → List String` (via map).

Different computational models are functors: Pure computation (Set) vs. I/O (IO type constructor) vs. stateful computation (State type constructor) are different "categories." Functors translate functions between them. This is why you can take a pure function and "lift" it into an effectful context.

How functors enable composition despite side effects:

  • Without functors: Side effects break composability. A function that does I/O can't be easily composed with a pure function.
  • With functors: The functor "lifts" pure functions into the effectful world. Now composition works again. The forgetful functor lets you go from effectful back to pure.
  • This is where monads enter: Monads are special functors that make this lifting and composition particularly elegant.

Intuition First

Think of a functor as a "translation" between categories. Just like a translator converts words from one language to another while preserving meaning, a functor maps objects and morphisms from one category to another while preserving structure. It's how you move from one mathematical "world" to another.

The Formal Definition

A functor is a structure-preserving map between categories. Formally, a covariant functor F : C → D consists of:

  • For every object A in C, an object F(A) in D.
  • For every morphism f : A → B in C, a morphism F(f) : F(A) → F(B) in D.

Structure preservation rules:

  • Composition preservation: F(g ∘ f) = F(g) ∘ F(f) — if two morphisms compose in C, their images compose in D the same way.
  • Identity preservation: F(id_A) = id_{F(A)} — identity morphisms map to identity morphisms.

Why these rules matter: Without them, the functor would scramble the structure it's supposed to translate. These rules guarantee that paths through C map to equivalent paths through D.

Examples

The Forgetful Functor: U : Vect_k → Set
Forgets that a vector space has extra structure and treats it as just a set. The functor maps each vector space V to its underlying set, and each linear map f to the same function (viewed as a set function).
The Fundamental Group: π₁ : Top* → Grp
Maps pointed topological spaces to groups (the fundamental group at the base point) and continuous maps to group homomorphisms.
The Powerset Functor: P : Set → Set
Maps each set X to its powerset P(X) (the set of all subsets). For a function f : X → Y, it sends each subset S ⊆ X to its image P(f)(S) = {f(x) | x ∈ S}.
Homology: H_n : Top → Ab (abelian groups)
Maps topological spaces to their n-th homology groups.

Composition of Functors

Functors can be composed. If F : C → D and G : D → E, then their composition G ∘ F : C → E is defined by:

  • (G ∘ F)(A) = G(F(A))
  • (G ∘ F)(f) = G(F(f))

Special Types of Functors

Faithful Functor

A functor F : C → D is faithful if it is injective on morphisms: whenever F(f) = F(g), we have f = g. This means the functor doesn't collapse distinct morphisms.

Full Functor

A functor F : C → D is full if it is surjective on morphisms: for every pair of objects A, B in C and every morphism h : F(A) → F(B) in D, there exists a morphism f : A → B in C such that F(f) = h.

Fully Faithful Functor

A functor that is both full and faithful. This is a very strong form of "sameness" between categories.

Equivalence of Categories

Two categories C and D are equivalent if there exist functors F : C → D and G : D → C such that G ∘ F ≅ Id_C and F ∘ G ≅ Id_D (up to natural isomorphism). Equivalent categories are "essentially the same" even if not identical.

Why Functors Matter

Functors allow us to:

  • Translate problems from one category to another.
  • Compare different mathematical structures.
  • Extract essential information while forgetting irrelevant details.
  • Build larger categories from smaller ones.
